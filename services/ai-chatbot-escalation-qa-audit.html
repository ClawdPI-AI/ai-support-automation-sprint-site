Audit deliverables

Concrete artifacts before implementation.

The audit is designed to produce a decision path, not a vague chatbot recommendation.

What you receive
  • ✓Restricted-topic map naming what the chatbot must not answer or must escalate.
  • ✓Human handoff rules for policy, safety, account, legal, billing, angry-customer, and unknown cases.
  • ✓QA prompt set for common questions, edge cases, missing-source cases, adversarial wording, and unclear intent.
  • ✓Failure logging plan for unknown questions, bad source coverage, and escalation misses.
  • ✓Launch recommendation for no launch, limited pilot, or implementation sprint.
Inputs needed
  • ✓Planned chatbot use cases, support/help-center URL, source documents, and current support stack.
  • ✓Known restricted topics, sensitive cases, support-policy owner, and approval owner.
  • ✓Existing vendor, chatbot, workflow, or platform plan if already selected.
  • ✓Any current QA prompts or acceptance criteria if available.
